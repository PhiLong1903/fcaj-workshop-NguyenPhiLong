**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting acquainted with the AWS Management Console interface, creating an AWS account](1.1-week1/)

**Week 2:** [Learning about AWS Identity and practicing labs on Users, Groups, and Roles](1.2-week2/)

**Week 3:** [Learning about VPC, EC2, subnets, security groups...](1.3-week3/)

**Week 4:** [Learning more about EC2 and how to optimize costs for virtual hosts](1.4-week4/)

**Week 5:** [Learning and practicing labs related to AWS RDS, starting to build the architecture for the team project](1.5-week5/)

**Week 6:** [Practicing deployment labs with auto-scaling and load balancer, AWS budget, continuing to upgrade and fix architecture errors](1.6-week6/)

**Week 7:** [Start building the group project and participate with the team in architecture development](1.7-week7/)

**Week 8:** [Learn about services integrated into the frontend, API Gateway, Lambda. Build some functionalities for the Project](1.8-week8/)

**Week 9:** [Describe the workflow of the architecture and continue building the group project](1.9-week9/)

**Week 10:** [Build the frontend integrating S3, Cloudfront, and WAF. Build interview-related functionalities for the project](1.10-week10/)

**Week 11:** [Responsive layouts. Build additional functionalities for the project](1.11-week11/)

**Week 12:** [Polish the UI, create a test project to prepare for the demo. Write user documentation](1.12-week12/)
